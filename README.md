# Bookmarklets
Original plan is to create Bookmarklets that will store a Job posting information locally for your job hunting reference.

I have been job hunting for over a year with a lot of ghost responses. So I have established standards that will help me capture information and be able to reference it at a future date.

## Long-term Goals
Extract information locally as the Job Posting often expires overtime. Capturing the information allows for ability to run AI on the extracted information from various postings that you have been interested in. Running all types of analytics and keyword usage across all files.

1. Create a .TXT file to be stored locally.
    - filename format: YYYYMMDD - Company Name__Job_Title.txt
    - example: 20260304 - S&P Global__Engineering.txt
        - YYYYMMDD is the current date
2. Extract information into Google Sheets to help track Application submission and Response information

## Notes
Bookmarklets are saved as .JS files which will not run correctly as a bookmarklet is wrapped in a "javascript:()" function. 

Please use with caution as I am running through many use cases but there may be items that haven't been played out. Attempting for speed.

This is an ongoing development where more will be added as time goes on. Come back soon.

## Bookmarklet List
1. Before you get started go to your Bookmark Manager.
2. Create a new folder with the name Bookmarklets.
3. Add a new bookmark to reside in this folder for each of the following:

### Greenhouse Job Posting
Say you found a job you're interested in applying to that is on a Greenhouse site.
URL example: https://job-boards.greenhouse.io/company_name/

1. Go to Greenhouse_Job_Posting.js and copy the code.
2. Create a bookmark with this code as the URL.
3. Give it the name "Greenhouse Job Posting".
4. While on the Job Posting page in the browser of your choice click on the bookmarklet.
